# Portfolio Operating System

## Stage 1: Research The Current Portfolio

- Read the current implementation, CV, and public links.
- Capture factual statements, proof points, and weak spots.
- Separate current reality from aspirational positioning.

## Stage 2: Normalize Content

- Standardize dates, titles, labels, and skill taxonomy.
- Convert long paragraphs into structured evidence.
- Identify which content belongs on the homepage versus deep pages.

## Stage 3: Redesign IA And Messaging

- Rebuild homepage hierarchy around recruiter scanning behavior.
- Clarify the Blue Team to Red Team narrative.
- Decide what proof needs to move higher on the page.

## Stage 4: Build The Visual System

- Establish typography, spacing, surface hierarchy, and color rules.
- Define component families and motion limits.
- Test hierarchy on mobile before polishing desktop.

## Stage 5: Implement Sections

- Rebuild section by section using structured content.
- Keep components maintainable and data-driven.
- Add route-level metadata and reusable utilities early.

## Stage 6: Audit And Refine

- Recruiter signal check
- Accessibility pass
- SEO metadata pass
- Performance pass
- Consistency audit

## Stage 7: Ship

- Re-run the definition of done.
- Remove weak or redundant UI.
- Ensure the portfolio is ready for future writeups, case studies, and certifications.
