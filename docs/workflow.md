# Workflow

## Canonical Workflow

1. Audit the current portfolio and CV.
2. Extract factual content into structured source-of-truth files.
3. Normalize terminology, dates, and taxonomy.
4. Redesign information architecture.
5. Redesign copy and hierarchy section by section.
6. Define visual system and design tokens.
7. Build or refactor sections in the app.
8. Run recruiter, accessibility, SEO, performance, and consistency audits.
9. Refine weak sections and remove clutter.
10. Ship only when the definition of done is met.

## Required Inputs Before Major Changes

- Current section content
- Relevant docs from docs/
- Applicable agent file from agents/
- Applicable skill from .cursor/skills/
- Relevant rules from .cursor/rules/

## Review Sequence

- Brand and positioning review
- UX and hierarchy review
- Copy review
- Engineering review
- Accessibility/SEO/performance review
- Final recruiter perspective review
