# Design Principles

## Principles

### 1. Lead With Clarity
The first screen must explain who Pablo is, what he does now, and where he is heading.

### 2. Make Evidence Easy To Find
Projects, labs, writeups, and certifications should work as proof, not decoration.

### 3. Premium Through Restraint
Use spacing, typography, contrast, and hierarchy to create sophistication. Avoid gimmicks.

### 4. Security-Native, Not Cartoon Security
The aesthetic can be dark-first and technical, but it must avoid hacker cliches, matrix effects, or novelty chrome.

### 5. Support Fast Scanning
Use short sections, meaningful labels, selective density, and clear grouping so recruiters can skim without losing substance.

### 6. Motion Must Earn Its Place
Animation should orient, reveal, or reinforce hierarchy. If it only decorates, remove it.

### 7. Accessibility Is Part Of Quality
Readable contrast, keyboard support, motion reduction, semantic structure, and touch-friendly layout are mandatory.

### 8. Build For Extension
The system must scale to future blog posts, writeups, case studies, speaking, certifications, or services.
