# Conventions

## Naming

- Use sentence case for UI labels unless a brand name requires otherwise.
- Use kebab-case for route segments, tasks, prompt files, and markdown file names.
- Use consistent names for repeated concepts: case study, writeup, skills matrix, certification path.

## Dates And Timeframes

- Use one date format across the site, preferably MMM YYYY - MMM YYYY or MMM YYYY - Present.
- Separate completed, in-progress, and planned items clearly.
- Never imply completion for a certification or project that is still planned.

## Content Formatting

- Prefer 1 to 2 sentence intros followed by scannable bullets or structured metadata.
- Keep tag systems normalized and deduplicated.
- Separate responsibilities, tools, and outcomes.

## Frontend Conventions

- Favor data-driven content structures over hard-coded repeated markup.
- Keep section components focused and composable.
- Avoid new dependencies unless they remove significant complexity.
- Default to server components unless client behavior is required.

## Quality Conventions

- Every substantive UX or copy change should have a rationale.
- Every new section should define purpose, audience, and CTA.
- Every public-facing claim should be traceable to the source-of-truth content.
