# UI System

## Desired Feel

- Dark-first if it improves contrast and brand fit.
- Premium, restrained, technical, and confident.
- More consultancy-grade than startup-marketing grade.

## Visual Vocabulary

- Large, calm type for headlines.
- Tighter measure and smaller supporting text for technical details.
- Card surfaces only when they improve grouping.
- Thin strokes, subtle borders, and selective glow used sparingly.
- Monospace accents only for metadata, tags, or utility labels.

## Color Strategy

- Neutral base dominates.
- Accent color should guide focus, not paint the whole interface.
- Purple/cyan can remain part of the identity if toned down and used intentionally.
- Avoid bright green as a primary signifier unless used with restraint for state or signal.

## Component Families

- Section wrappers
- Hero and trust modules
- Timeline or experience cards
- Skills taxonomy panels
- Project/case-study cards
- Metric or evidence chips
- CTA blocks
- Navigation and footer systems

## Motion Guidelines

- Use fade, translate, and subtle scale.
- Keep durations tight and predictable.
- Respect prefers-reduced-motion.
- Avoid perpetual ambient motion except where it communicates state.

## Token Priorities

- Spacing scale
- Type scale
- Border radius
- Surface hierarchy
- Accent usage rules
- Shadow and glow restraint
