# Subagents

## Hero-Section-Subagent
- Mission: sharpen first-screen clarity, trust, and CTA quality.
- Use when: the hero feels vague, noisy, or generic.
- Outputs: headline options, supporting line, trust modules, CTA rationale.
- Done when: role, direction, and next action are obvious in seconds.

## About-Section-Subagent
- Mission: convert biography into a concise strategic narrative.
- Use when: the about section is too long or unfocused.
- Outputs: narrative arc, proof bullets, rewrite, rationale.
- Done when: differentiation is clear without over-explaining.

## Experience-Section-Subagent
- Mission: turn experience into recruiter-readable evidence.
- Use when: role entries feel dense or tool-heavy.
- Outputs: role summaries, proof bullets, skill tags, hierarchy notes.
- Done when: relevance to target roles is easy to scan.

## Certifications-Subagent
- Mission: present certifications and education as trajectory and validation.
- Use when: certifications feel decorative or ambiguous.
- Outputs: status wording, hierarchy, supporting labels.
- Done when: current, planned, and completed states are unmistakable.

## Skills-Matrix-Subagent
- Mission: normalize the skills taxonomy so it signals depth rather than keyword stuffing.
- Use when: skills are repetitive or overly broad.
- Outputs: categories, merges, removals, ordering.
- Done when: the matrix is clean, credible, and easy to parse.

## Projects-Subagent
- Mission: improve project cards and archive structure.
- Use when: project value is buried or generic.
- Outputs: card structure, metadata rules, stronger summaries.
- Done when: each project explains challenge, method, and relevance.

## Writeups-CTF-Subagent
- Mission: shape labs and writeups into proof of practical offensive-security progress.
- Use when: writeups feel isolated from the broader story.
- Outputs: section strategy, labels, trust framing, future model.
- Done when: practical practice supports the career narrative.

## Contact-CTA-Subagent
- Mission: improve conversion pathways without sounding desperate or generic.
- Use when: contact feels like a default contact page.
- Outputs: CTA copy, role-fit framing, trust improvements.
- Done when: the page feels professional and action-oriented.

## Mobile-UX-Subagent
- Mission: protect hierarchy and usability on smaller screens.
- Use when: desktop assumptions dominate the design.
- Outputs: spacing fixes, order changes, touch and scan recommendations.
- Done when: mobile feels intentional, not compressed desktop.

## Accessibility-Subagent
- Mission: check semantic, contrast, motion, and interaction quality.
- Use when: a section is visually polished or recently refactored.
- Outputs: severity-ranked accessibility findings and fixes.
- Done when: blockers are removed and key risks are documented.

## Design-Tokens-Subagent
- Mission: translate visual direction into reusable token decisions.
- Use when: the UI lacks consistency across sections.
- Outputs: spacing, type, color, radius, surface, and motion token guidance.
- Done when: token decisions can drive implementation consistently.

## Visual-Hierarchy-Subagent
- Mission: improve section emphasis, density, and scanning rhythm.
- Use when: everything competes for attention.
- Outputs: priority map, layout changes, removals, rationale.
- Done when: the page feels calm, ordered, and premium.
